## 1. Business purpose
<img width="400" alt="Image" src="https://github.com/user-attachments/assets/dd25a45f-75d2-47df-9caa-561010a930c3" />
<img width="400" alt="Image" src="https://github.com/user-attachments/assets/9c45cdd4-7342-4d0b-93b6-46b53ab1da77" />
<br>
2025년 해킹으로 인한 국내/국외 대형 통신사 및 대기업들의 고객 데이터 유출 사건이 크게 문제가 되었다. 기업 내부 차원에서 내부망에 대한 비정상적 접근을 탐지 하지 못했고 노후화된 서버 구조나 클라우드 기반 서비스의 보안 설정, 접근 권한 관리 등등 다양한 원인이 문제가 되었다. 또 유출된 정보를 바탕으로 크리덴셜 스터핑 등의 공격을 통해 고객이 가입한 다른 사이트들이 연쇄적으로 공격당하는 경우도 크게 늘었다. 특히 크리덴셜 스터핑 공격은 보통 사용자들이 다른 사이트들에 대해 같은 비밀번호를 돌려쓰는 경우 치명적이다.

그렇다면 서비스 이용자들은 이러한 해킹 공격들에 대해 어떻게 대처해야 할까? 가장 간단하고 좋은 방법은 사이트마다 비밀번호를 다르게 하고 규칙성 없는 무작위로 설정하며 주기적으로 변경하는 것이다. 하지만 서비스 이용자들은 수많은 사이트에 회원가입하고 패턴이 없는 비밀번호를 일일이 다 기억하기 어려우며 그 많은 비밀번호를 주기적으로 변경하는 것도 쉽지 않다. 통계 자료에서도 볼 수 있듯 6개월 이상 비밀번호를 변경하지 않는 사용자들이 대부분이고 세계에서 가장 많이 사용되는 비밀번호가 ‘123456’이다. 실제로 같은 비밀번호를 돌려쓰면서 1년 이상 비밀번호를 바꾸지 않아 네이버 계정을 해킹 당해본 경험도 있었다.

이런 경험들을 바탕으로 비밀번호를 자동으로 생성 해주고 내가 만든 계정들을 한 번에 모아서 관리해 줄 수 있는 서비스가 있으면 좋겠다고 생각해 본 프로젝트를 구상하게 되었다. 시중에는 클라우드 서버를 기반으로 한 비밀번호 관리 프로그램들이 있고 편의성 또한 좋지만 서버 데이터 유출이나 외부 공격에 대한 잠재적인 위험이 존재한다. 또한 로컬 시스템으로 만들어진 비밀번호 관리 프로그램들은 사용자의 Device에 종속적이어서 비밀번호 관리 프로그램이 설치되어 있지 않은 다른 Device로는 비밀번호 데이터베이스에 접근하기 어렵다.

본 프로젝트의 ‘Masterkey’ 시스템은 로컬 시스템으로 만들되 USB/외장 HDD 등에 이식해 비밀번호 관리 프로그램이 설치되어 있지 않은 Device에서도 작동할 수 있게 만들 것이다. 프로그램을 사용자 device에 설치할 경우 일반 로컬 시스템과 동일하게 작동하게 할 것이다. 또한 자동 백업 기능과 비밀번호 자동 입력 기능을 통해 기존의 프로그램들이 수동으로 해줘야 했던 작업을 자동화로 만들 것이다.

**본 프로젝트의 목표는 다음과 같다.**
1. 사용자의 모든 비밀번호를 하나의 암호화된 저장소에 안전하게 저장한다.
2. 강력한 비밀번호를 자동으로 생성하고 보안 수준을 분석해 사용자의 보안 수준을 향상시킨다.
3. 비밀번호 변경 주기 관리 및 일괄 변경 기능을 통해서 지속적인 보안 관리를 지원한다.
4. USB/외장 HDD 기반 휴대형 구조를 통해 언제 어디서든 동일한 환경에서 비밀번호를 관리할 수 있도록 한다.

**본 프로젝트의 주 타겟층은 다음과 같다.**
1. 여러 사이트에서 각기 다른 비밀번호를 쓰지만 관리가 어려운 일반 사용자.
2. 다수의 서버 혹은 서비스 계정을 가지고 있어야 하고 높은 계정 보안을 요구하는 개발자 및 IT/보안 관련 직업군.
3. 수많은 고객 계좌를 다루기 위해서 고객 계정을 안전하게 관리해야 하는 금융/재무 관련 직업군 혹은 업무용 계정과 개인 개정을 따로 관리하고 싶어 하고 동시에 보안 의식이 높은 사용자.

## 4. Concept of operation

### 1) Exit/Master Password input
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 암호화된 데이터베이스에 접근하기 위해서 마스터 비밀번호를 입력. 반대로 사용자가 이용을 완료하면 다시 데이터베이스를 암호화. |
| **Approach** | 사용자가 암호화된 데이터베이스에 접근하여 저장한 계정 비밀번호를 복호화 하기 위해서 프로그램을 실행시키거나 데이터베이스 파일을 더블 클릭 하면 사용자의 데이터베이스가 맞는지 확인하기 위해 마스터 비밀번호를 입력창을 보여준다. 만약 데이터베이스가 저장된 Device에 사용자가 아닌 다른 사람이 그 파일에 접근하려고 해도 마스터 비밀번호를 모르면 데이터베이스에 접근하지 못하도록 한다. 반대로 프로그램을 Exit 하면 다시 마스터 비밀번호 입력창으로 이동한다. |
| **Dynamics** | 프로그램을 실행하고 데이터베이스를 생성할 시에 마스터 비밀번호 입력창을 준다. 이 후 데이터베이스에 접근 시 마스터 비밀번호로 사용자임을 인증한다. |
| **Goals** | 데이터베이스의 암호화 복호화를 진행한다. |

### 2) Create database
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 계정 정보와 비밀번호 정보를 저장하기 위해. |
| **Approach** | 프로그램을 실행하면 제일 먼저 데이터베이스를 새로 만들지 불러올지 결정하게 한다. 사용자가 데이터베이스 파일을 새로 만든다면 저장할 경로를 정하게 한다. 만약 불러오는 것을 선택할 경우 파일을 선택하게 한다. |
| **Dynamics** | 프로그램을 실행할 시 데이터베이스를 새로 만들기를 선택하는 경우. |
| **Goals** | 비밀번호를 저장할 데이터베이스를 생성한다. |

### 3) Set / Change Master Password
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | Exit/Master Password input 기능을 동작시키기 위해. |
| **Approach** | 데이터베이스를 새로 생성할 시 그 데이터베이스에 대한 마스터 비밀번호 입력창을 보여준다. 혹은 이미 만들어진 데이터베이스의 마스터 비밀번호를 변경하기를 선택한다. 추가로 복구키도 생성한다. |
| **Dynamics** | 데이터베이스를 새로 생성하는 경우 사용자로부터 입력. |
| **Goals** | 마스터 비밀번호를 설정하고 변경한다. |

### 4) Manage Account Entries
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 데이터베이스의 내용을 조작하기 위해. |
| **Approach** | 마스터 비밀번호를 통해 사용자 인증 후 데이터베이스 파일에 접근하고 이후 계정과 비밀번호를 추가, 조회, 수정, 삭제한다. |
| **Dynamics** | 데이터베이스에 접근 후 비밀번호를 추가, 수정, 삭제, 조회 하는 경우. |
| **Goals** | 데이터베이스의 내용을 사용자가 조작하도록 한다. |

### 5) Load Database
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 동시에 여러 개의 데이터베이스에 접근하기 위해. |
| **Approach** | 프로그램 창에서 어떤 데이터베이스에 접근하고 동시에 다른 데이터베이스에 접근할 수 있도록 데이터베이스 불러오기를 선택한다. |
| **Dynamics** | 프로그램 창에서 데이터베이스 불러오기를 선택할 경우. |
| **Goals** | 동시에 여러 개의 데이터베이스 정보를 볼 수 있게 한다. |

### 6) Backup Database / Auto Backup
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | USB나 외장 HDD에 프로그램과 데이터베이스를 저장하여 다른 Device에서도 사용할 경우 USB/HDD와 Device 사이의 데이터베이스의 동기화를 위해. 또는 데이터베이스 파일을 유실할 경우 백업을 위해. |
| **Approach** | 프로그램 창에서 데이터베이스에 접근하고 백업 옵션을 선택한다. 데이터베이스 파일이 수정되면 백업 경로에 따라 자동으로 백업을 진행한다. |
| **Dynamics** | 백업 옵션을 선택할 경우. |
| **Goals** | 백업을 진행하기 위해. |

### 7) Generate Password
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 비밀번호 보안 강도를 높이기 위해. |
| **Approach** | 데이터베이스에 계정 비밀번호를 추가하기 위해서 자동으로 생성한다. 사용자는 문자, 특수문자, 번호, 길이 등을 선택한다. |
| **Dynamics** | 데이터베이스에 비밀번호를 추가하는 경우. |
| **Goals** | 무작위 비밀번호로 보안 수준을 높인다. |

### 8) Bulk Password Update
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 비밀번호 일괄 변경을 위해. |
| **Approach** | 프로그램 창에서 일괄 변경 선택한다. |
| **Dynamics** | 비밀번호를 일괄 변경하고 싶은 경우. |
| **Goals** | 비밀번호를 전체 변경 처리한다. |

### 9) Copy Password/ Autofill Password in Browser
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 비밀번호 입력을 돕기 위해. |
| **Approach** | 데이터베이스 접근 후 사용자의 계정 목록에서 클립보드로 복사를 선택. 또는 자동 입력을 선택. 클립보드 복사의 경우 10초 후 클립보드 비우기 실행. |
| **Dynamics** | 사용자가 데이터베이스에 저장된 비밀번호를 사이트에 입력할 경우. |
| **Goals** | 긴 비밀번호를 직접 입력하지 않고 복사할 수 있게 하여 편의성을 제공한다. |

### 10) Password History & Last Change Alert
| 항목 | 내용 |
| :--- | :--- |
| **Purpose** | 비밀번호 지속 관리를 지원하기 위해. |

## 5. Problem statement

### problem 1: “마스터 비밀번호 하나에 모든 보안을 의존”
- 본 프로젝트에서 구상한 비밀번호 관리 프로그램은 데이터베이스 파일과 그에 대한 키만 알고 있으면 모든 계정의 비밀번호에 대해 접근할 수 있게 된다. 즉, 마스터 비밀번호를 어떻게 관리하냐에 따라서 보안 위협이 크게 달라진다. 사용자에게 이 점을 알리고 비밀번호가 얼마나 안전한지에 대한 정보를 제공하여 사용자가 경각심을 가지고 비밀번호를 관리할 수 있도록 해야 한다. 또한 복구키는 꼭 USB/HDD 등 외부 장치에서 관리할 수 있도록 사용자에게 알린다.

### problem 2: “데이터베이스 파일에 대한 암호 알고리즘”
- 비밀번호 관리 프로그램에서 데이터베이스 파일은 사용자의 개인정보가 암호화되어 직접적으로 저장되는 아주 중요한 파일이다. 여기서 이미 보안 취약점이 알려진 보안 알고리즘, 또는 너무 간단한 알고리즘을 통해 암호화를 진행할 경우 고객들의 개인정보가 매우 위험해진다. 안전하게 암호화하여 데이터를 보관할 수 있는 알고리즘을 배우고 구현할 수 있도록 프로젝트 진행 간 알고리즘 공부를 할 것이다.

### problem 3: “USB/HDD와 같은 데이터베이스를 저장한 장치의 분실 위험”
- 만약 사용자의 PC에 데이터베이스를 백업해 두지 않고 USB/HDD와 같은 이동식 장치에만 데이터베이스를 저장한다면 이동식 장치 분실 시 다시는 데이터베이스에 접근할 수 없으므로 모든 사이트에 대해서 수동으로 비밀번호를 다시 찾고 변경해야 한다. 이 때문에 견고한 백업 시스템을 제공해야 한다. 데이터베이스를 백업할 수 있도록 사용자에게 경고하고 자동으로 백업을 할 수 있는 방법을 찾아 제공할 것이다.

### problem 4: “과도한 기능 추가와 GUI의 복잡성에 대한 위험”
- 비밀번호 관리 프로그램 특성상 보안이 중요하지만, 과도한 편의성 기능을 추가 시 설계자도 생각하지 치명적인 문제가 생길 수 있기에 API 및 특별한 자동화 시스템 등등은 최대한 줄여 취약점을 보완해야 한다. 하지만 보안 강도를 위해 편의성을 과도하게 줄이고 보안 옵션에 대한 복잡한 GUI를 사용할수록 사용자는 프로그램을 더 사용하기 어려워진다. 결국 사용자는 프로그램 사용 숙지 미숙으로 보안 사항들을 그냥 무시할 수 있기에 사용자 친화적 GUI 및 편의성 기능과 보안 강도 사이에 적당한 타협점을 찾아야 한다. 본 프로젝트에서는 복잡한 기능과 다양한 보안 옵션 제공보다는 누구나 쉽게 사용할 수 있도록 단순하고 사용자 친화적인 인터페이스를 사용할 것이다.

## 6. Glossary

| term | Description |
| :--- | :--- |
| **크리덴셜 스터핑** | 이미 유출된 계정 정보(아이디/비밀번호)를 이용하여 여러 서비스에 자동으로 로그인 시도를 하는 공격 방식. |
